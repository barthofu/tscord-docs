# Configuration

import Tabs from '@theme/Tabs'
import TabItem from '@theme/TabItem'

TSCord supports different databases:
- SQLite (by default)
- MySQL
- MariaDB
- PostgreSQL

:::tip
By default, the template is configured to use **SQLite**. If your bot doesn't aim to be used on thousands of guilds with 
hundred of thousands of users, you should keep it.

Plus, SQLite is the only database supported by the **[automatic backup system](/docs/bot/features/database/backups)**.
:::

You can configure all of this in the `src/config/database.ts` file.

Indeed, this file exports a a `mikroOrmConfig` constant, which contains a production configuration and an optional development configuration. At runtime, the good configuration is chosen thanks to the NODE_ENV environment variable.
The database type should be explicitely set in the `mikroOrmConfig.production.type` constant as it will be used to determine which entity manager type Mikro-ORM should use internaly.

You can find here the configuration for the different types of databases:

<Tabs>
<Tab label="SQLite" value="sqlite">

```ts title=mikroOrmConfig
{
    type: 'sqlite',
    dbName: `${databaseConfig.path}db.sqlite`, // path to the db .sqlite file
}
```
</Tab>

<Tab label="PostgreSQL" value="postgresql">

```ts title=mikroOrmConfig
{
    type: 'postgres',
    dbName: process.env['DATABASE_NAME'],
    host: process.env['DATABASE_HOST'],
    port: 5432,
    user: process.env['DATABASE_USER'],
    password: process.env['DATABASE_PASSWORD'],
}
```
</Tab>

<Tab label="MySQL" value="mysql">


## Env 

You surely noticed that the database connection informations are stored in the environment variables.

You can edit it in the `.env` file.

Example:

```python title=.env
#...

DATABASE_HOST="database"
DATABASE_PORT=3306
DATABASE_NAME="tscord_bot"
DATABASE_USER="tscord"
DATABASE_PASSWORD="tscord123"

#...
```

## Better-sqlite3

The `better-sqlite3` is a way better package than the default `sqlite3` package. It's faster and more reliable. You can have more informations on this [there](https://github.com/WiseLibs/better-sqlite3/issues/262).

Its main downside though is that it relies on native bindings via `node-gyp`, which means that you need to have a correct native build tools installed on your system to be able to install it. Before the `v2.4`, it was the default database of TSCord but we decided to switch to the default `sqlite3` package to avoid any frequent installation issues to newcomers.

But if you know how to install node-gyp well and you want to use `better-sqlite3`, you can install it by following these steps:
1. `npm i @mikro-orm/better-sqlite`
2. `npm i -D @types/better-sqlite3`
3. Add the following lines in `src/utils/types/database.d.ts`:
```ts
type DatabaseConfigs = {
	sqlite: {
		driver: import('@mikro-orm/sqlite').SqliteDriver
		entityManager: import('@mikro-orm/sqlite').SqlEntityManager
	}
//highlight-start
    'better-sqlite': {
		driver: import('@mikro-orm/better-sqlite').BetterSqliteDriver
		entityManager: import('@mikro-orm/better-sqlite').SqlEntityManager
	}
//highlight-end
	postgresql: {
		driver: import('@mikro-orm/postgresql').PostgreSqlDriver
		entityManager: import('@mikro-orm/postgresql').SqlEntityManager
	}
    // ...
}
```
4. Change the database `type` to `better-sqlite` in the `src/configs/database.ts` file.
